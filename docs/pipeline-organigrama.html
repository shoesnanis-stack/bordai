BordAI — Organigrama del Pipeline

Flujo completo desde que el cliente sube su imagen hasta que descarga el archivo de bordado

F0
ONBOARDING Usuario
El cliente configura su proyecto
Quien trabaja

El cliente en el frontend. No hay IA ni Python aqui.

Que hace
  • Elige superficie (gorra, camisa, chamarra, parche)
  • Elige tamano de aro (4x4, 5x7, 6x10...)
  • Elige maquina (Tajima, Brother, Janome...)
  • Sube imagen del diseno
Que produce

Imagen guardada en Supabase Storage. Registro en tabla project_files tipo "original".

Avanza a F1

F1
COMPRENSION GPT-4o Vision
La IA analiza la imagen y dice que entendio
Quien trabaja
Agente: GPT-4o Vision
Recibe la imagen + config del proyecto. Le dice al cliente "Entendi que quieres bordar un logo de LA en negro sobre camisa".
Que hace
  • Mira la imagen con vision
  • Identifica elementos (logo, texto, fondo...)
  • Dice que mantener y que descartar
  • Evalua calidad (good/poor/unusable)
  • Estima cuantos colores de hilo
  • Evalua complejidad
Que produce

Brief guardado en tabla briefs:

  • Texto de lo que entendio
  • Lista: mantener / descartar
  • Calidad + complejidad
  • N colores estimados

El cliente aprueba o re-analiza

F2
EXTRACCION Automatico / Skip
Separar el diseno del fondo (pendiente SAM)
Quien trabaja

Nadie todavia — el cliente da click en "Continuar" y se salta este paso.

FUTURO: SAM (segmentacion IA) + Real-ESRGAN (mejora calidad)

Que hace
  • Copia la imagen original como "extracted"
  • Marca skipped_extraction: true
Que produce

Registro en project_files tipo "extracted" (misma imagen).

Avanza a F3

F3
PREPARACION GPT-4o Vision
La IA asigna puntadas, colores y angulos a cada region
Quien trabaja
Agente: GPT-4o Vision (Digitalizador)
Recibe la imagen + tamano del aro. Actua como digitalizador profesional: decide que tipo de puntada va en cada parte del diseno.
Que hace
  • Divide el diseno en regiones
  • Asigna tipo: tatami / satin / running
  • Asigna color hex REAL de los pixeles
  • Asigna densidad (4-6 st/mm)
  • Asigna angulo de relleno
  • Decide si lleva underlay
  • Calcula dimensiones en mm
Que produce

JSON guardado en project_files metadata tipo "vectorized":

  • Array de regiones con params
  • Lista de colores de hilo
  • Estimacion de puntadas
  • Dimensiones width x height mm

El cliente ve la tabla y avanza

F4
VISTA PREVIA Usuario
El cliente revisa y aprueba antes de generar
Quien trabaja

El cliente revisa el preview. No hay IA ni Python.

Que ve
  • Imagen original
  • Tabla de regiones + colores
  • Tamano real en mm
  • N colores de hilo
  • Boton "Aprobar"
Que produce

Aprobacion del cliente. El proyecto avanza a F5.

Sin aprobacion NO se genera

F5
GENERACION Python Engine v4
El motor genera el archivo de bordado real (.DST / .PES / .JEF)
Quien trabaja
Agente: Python (generate_embroidery.py v4)
Recibe imagen + parametros de F3 + perfil de tela. Genera las coordenadas exactas de cada puntada que la maquina va a coser.
Sub-agente: Perfil de Tela (perfiles.py)
Dice los parametros segun la superficie: densidad, compensacion, tipo de underlay, umbral de corte, estabilizador recomendado.
Pasos internos del Python
1
Carga perfil — shirt/cap/jacket/patch
2
Lee imagen del archivo temporal
3
Crea mascara — separa diseno del fondo (alpha / binario / bordes)
4
Preserva huecos — detecta agujeros internos (ej: hueco de la A)
5
Separa colores — crea mascara por region/color
6
Ordena — fills primero, luego satins, luego contornos. Claros antes que oscuros
7
Por cada region:
a. COLOR_BREAK si cambia hilo
b. TIE-IN (3 puntadas de seguro)
c. UNDERLAY (zigzag+edge / edge / center segun perfil)
d. PUNTADAS PRINCIPALES (scanline + angulo + pull compensation)
e. CONTORNO (running stitch en borde)
f. TIE-OFF (3 puntadas de cierre)
8
TRIM — corta hilo en saltos > 3mm
9
Valida — cuenta puntadas, verifica aro, colores
10
Exporta — pyembroidery escribe .DST/.PES/.JEF
Que produce
  • Archivo .DST/.PES/.JEF subido a Storage
  • Debug logs (mascara, regiones, puntadas)
  • Metadata con stats completas

Panel de diagnostico visible en UI

El cliente ve: metodo de mascara, pixeles por region, total de puntadas, log de Python.

F6
ENTREGA Usuario
El cliente descarga su archivo de bordado
Quien trabaja

El cliente descarga. Solo se genera una URL firmada de Supabase Storage.

Que hace
  • Genera URL firmada (1 hora)
  • Descarga el archivo al navegador
  • Nombre: {proyecto}.{formato}
Que produce

Archivo listo para cargar en la maquina de bordado.

FIN DEL PIPELINE

Resumen de "Empleados"

Empleado Rol Donde trabaja Fases
Cliente Sube imagen, configura, aprueba, descarga Navegador (Next.js frontend) F0, F1, F2, F4, F6
GPT-4o Vision Analista Mira la imagen, genera brief para el cliente API route /analyze (OpenAI) F1
GPT-4o Vision Digitalizador Asigna puntadas, colores, angulos, densidad API route /prepare (OpenAI) F3
Python Engine Motor v4 Genera el archivo de bordado real con puntadas generate_embroidery.py (child_process) F5
Perfil de Tela Define parametros por tipo de superficie perfiles.py (importado por motor) F5
Supabase Base de datos Guarda todo: proyectos, archivos, briefs, metadata PostgreSQL + Storage (nube) Todas
